# 贪心算法（Greedy Algorithm）

## 一、概念与定义

贪心算法是一种在每一步决策时都选择**当前状态下的局部最优解**，从而希望最终达到**全局最优解**的算法策略。其核心思想是：

> **不回溯、不考虑未来**——在每个决策点做出当下看起来最好的选择，并且一旦做出选择就不再改变。

形式化描述：对于一个优化问题，贪心算法将问题分解为一系列子步骤，在每一步中选择使目标函数增量最大（或代价最小）的选项，直到问题求解完毕。

## 二、贪心算法的关键性质

贪心算法能保证得到全局最优解，需要问题同时满足以下两个性质：

| 性质             | 含义                                                                         |
| ---------------- | ---------------------------------------------------------------------------- |
| **贪心选择性质** | 通过局部最优选择，能够产生全局最优解；即每一步的最优选择不会排除掉全局最优解 |
| **最优子结构**   | 问题的最优解包含其子问题的最优解                                             |

如果问题不满足这两个性质，贪心算法可能只能得到**近似解**而非最优解。

## 三、算法一般步骤

1. **建模**：明确问题的优化目标和约束条件
2. **确定贪心策略**：找到每一步的"局部最优"准则
3. **证明正确性**：证明贪心选择性质和最优子结构（交换论证法、归纳法等）
4. **迭代求解**：按贪心策略逐步构造解，直到问题解决

## 四、经典应用场景

### 1. 活动选择问题（Activity Selection）

**场景**：给定若干活动的开始和结束时间，选出最多互不重叠的活动。

**贪心策略**：每次选择**结束时间最早**且与已选活动不冲突的活动。

### 2. 霍夫曼编码（Huffman Coding）

**场景**：根据字符出现频率构造最优前缀编码，使总编码长度最短。

**贪心策略**：每次合并频率**最小**的两个节点。

### 3. 最小生成树

- **Kruskal 算法**：按边权从小到大排序，依次选边，跳过成环的边
- **Prim 算法**：从任一顶点出发，每次选与已有树相连的**最小权边**

### 4. 单源最短路径（Dijkstra 算法）

**场景**：在非负权图中求从源点到其他所有点的最短路径。

**贪心策略**：每次从未确定的节点中选择距离源点**最近**的节点进行松弛。

### 5. 分数背包问题（Fractional Knapsack）

**场景**：物品可以被切分，在容量限制下使总价值最大。

**贪心策略**：按**单位重量价值**从高到低装入。

> 注意：**0-1 背包**（物品不可切分）不满足贪心选择性质，需用动态规划求解。

### 6. 区间调度 / 区间覆盖

- **区间调度最大化**：选最多不重叠区间 → 按右端点排序
- **区间覆盖**：用最少区间覆盖目标 → 每次选能覆盖最远的区间

### 7. 跳跃游戏（Jump Game）

**场景**：数组中每个元素表示最大跳跃步数，判断能否到达终点/求最少跳跃次数。

**贪心策略**：维护当前能到达的最远位置，逐步更新。

### 8. 任务调度 / 带截止时间的作业排序

**场景**：每个任务有利润和截止时间，在截止时间内完成以获得最大利润。

**贪心策略**：按利润降序排列，尽量在最晚可用时间槽完成高利润任务。

## 五、什么场景适合用贪心？

适合使用贪心的**判断标准**：

1. **问题可分解为一系列选择步骤**，且每步的选择缩小问题规模
2. **局部最优 → 全局最优**：能证明（或直觉上）每步的最优选不会影响最终最优性
3. **无后效性**：当前选择不依赖于未来的选择结果
4. **问题具有最优子结构**

**典型适用场景总结**：

| 场景类型         | 典型问题                       |
| ---------------- | ------------------------------ |
| 区间/时间安排    | 活动选择、会议室分配、合并区间 |
| 图论优化         | 最小生成树、最短路径、网络流   |
| 编码/压缩        | 霍夫曼编码                     |
| 资源分配（可分） | 分数背包、加油站问题           |
| 字符串构造       | 字典序最小/最大的排列          |
| 找零/面额问题    | 特定面额体系下的最少硬币数     |

## 六、贪心 vs 动态规划

| 维度       | 贪心                                | 动态规划                           |
| ---------- | ----------------------------------- | ---------------------------------- |
| 选择方式   | 每步取局部最优，不回溯              | 考虑所有子问题，自底向上或带备忘录 |
| 时间复杂度 | 通常更低（$O(n \log n)$ 或 $O(n)$） | 通常更高（$O(n^2)$ 或更多）        |
| 适用条件   | 贪心选择性质 + 最优子结构           | 重叠子问题 + 最优子结构            |
| 正确性     | 需要严格证明                        | 天然保证最优（穷举子问题）         |

**经验法则**：先尝试贪心，如果无法证明贪心选择性质（或能举出反例），则考虑动态规划。


# 贪心算法——计算机视角的数学描述

## 一、问题建模

给定：
- 输入规模为 $n$ 的问题实例
- 候选集合 $C = \{c_1, c_2, \dots, c_n\}$
- 可行性判定函数 $\text{feasible}(S) \to \{true, false\}$，判断当前解集 $S$ 是否合法
- 目标函数 $\text{cost}(S) \to \mathbb{R}$，需要最大化或最小化

目标：找到 $S^* \subseteq C$，使得 $\text{feasible}(S^*) = true$ 且 $\text{cost}(S^*)$ 最优。

## 二、贪心算法伪代码

```
GREEDY(C, feasible, select):
    S ← ∅                          // 解集初始化为空
    candidates ← sort(C, by=select) // 按贪心策略排序

    for each x in candidates:
        if feasible(S ∪ {x}):       // 加入 x 后仍可行
            S ← S ∪ {x}            // 选入 x，不可撤销

    return S
```

核心三要素：

| 要素       | 作用                           | 举例（活动选择）       |
| ---------- | ------------------------------ | ---------------------- |
| `select`   | 贪心排序准则，决定考察顺序     | 按结束时间升序         |
| `feasible` | 可行性检查，加入后是否违反约束 | 新活动与已选活动不重叠 |
| 不可撤销性 | 一旦选入/排除，后续不再改变    | —                      |

## 三、时间复杂度分析

贪心算法通常由两部分组成：

$$
T(n) = T_{\text{sort}} + T_{\text{scan}}
$$

- **排序阶段**：将候选元素按贪心准则排序，通常 $O(n \log n)$
- **扫描阶段**：遍历排序后的元素，逐个判断是否加入解集，通常 $O(n)$（若 feasible 判断为 $O(1)$）

因此大多数贪心算法的总复杂度为 $O(n \log n)$，相比动态规划的 $O(n^2)$ 或更高，效率优势明显。

## 四、正确性证明的两种常用方法

### 方法一：交换论证（Exchange Argument）

**思路**：假设最优解 $O$ 与贪心解 $G$ 不同，通过逐步将 $O$ 中的元素"交换"为 $G$ 中的元素，证明替换后解不会变差。

```
证明框架：
1. 设 G = {g1, g2, ..., gk} 为贪心解，O = {o1, o2, ..., om} 为最优解
2. 找到第一个 G 和 O 不同的位置 i
3. 证明：将 O 中的 oi 替换为 gi，得到 O'，且 cost(O') ≤ cost(O)
4. 归纳：反复替换，最终 O 变成 G，说明 cost(G) = cost(O)
```

### 方法二：贪心保持领先（Greedy Stays Ahead）

**思路**：证明贪心算法在每一步的"进度"都不落后于任何最优解。

```
证明框架：
1. 定义一个度量 measure(Sk)，表示前 k 步的解质量
2. 用归纳法证明：对所有 k，measure(Gk) ≥ measure(Ok)
3. 最终 measure(G) ≥ measure(O)，即贪心解不劣于最优解
```

## 五、经典问题的计算机语言建模

### 1. 活动选择

```
输入：activities[] = {(s1,f1), (s2,f2), ..., (sn,fn)}  // (开始, 结束)
排序：按 fi 升序
贪心：
    last_end ← 0
    count ← 0
    for each (si, fi) in sorted_activities:
        if si ≥ last_end:        // 不与已选活动冲突
            选入该活动
            last_end ← fi
            count++
    return count
```

- `select`：按结束时间 $f_i$ 升序
- `feasible`：$s_i \geq \text{last\_end}$
- 复杂度：$O(n \log n)$

### 2. 分数背包

```
输入：items[] = {(vi, wi)}, 背包容量 W    // vi=价值, wi=重量
排序：按 vi/wi 降序（单位价值）
贪心：
    remain ← W
    total_value ← 0
    for each (vi, wi) in sorted_items:
        take ← min(wi, remain)   // 能取多少取多少
        total_value += vi * (take / wi)
        remain -= take
        if remain == 0: break
    return total_value
```

- `select`：单位价值 $v_i / w_i$ 降序
- 物品可以取分数部分 → 贪心最优
- 若物品不可分（0-1 背包）→ 贪心**不**最优，需要 DP

### 3. Huffman 编码

```
输入：chars[] 及其频率 freq[]
数据结构：最小堆（优先队列）

BUILD_HUFFMAN(chars, freq):
    pq ← 将所有字符按频率插入最小堆     // O(n)
    while pq.size() > 1:
        left ← pq.extractMin()          // O(log n)
        right ← pq.extractMin()         // O(log n)
        node ← new InternalNode(freq = left.freq + right.freq)
        node.left ← left
        node.right ← right
        pq.insert(node)                 // O(log n)
    return pq.extractMin()  // 根节点即 Huffman 树
```

- `select`：每次取频率最小的两个节点
- 复杂度：$O(n \log n)$
- 贪心本质：高频字符短编码，低频字符长编码，使加权路径长度 $\sum f_i \cdot d_i$ 最小

### 4. Dijkstra 最短路径

```
输入：图 G = (V, E)，权重函数 w(u,v) ≥ 0，源点 s
数据结构：最小堆

DIJKSTRA(G, s):
    dist[] ← ∞ for all v; dist[s] ← 0
    visited[] ← false
    pq ← {(0, s)}                       // (距离, 节点)

    while pq is not empty:
        (d, u) ← pq.extractMin()        // 贪心：取距离最小的未确定节点
        if visited[u]: continue
        visited[u] ← true
        for each neighbor v of u:
            if dist[u] + w(u,v) < dist[v]:
                dist[v] ← dist[u] + w(u,v)   // 松弛操作
                pq.insert((dist[v], v))
    return dist[]
```

- `select`：$\arg\min_{v \notin \text{visited}} \text{dist}[v]$
- `feasible`：边权非负 $w(u,v) \geq 0$（负权则贪心失效）
- 复杂度：$O((V + E) \log V)$（二叉堆实现）

## 六、贪心失效的判断

以下信号通常意味着贪心**不适用**，需要考虑 DP 或其他方法：

| 信号                                 | 示例                                                  |
| ------------------------------------ | ----------------------------------------------------- |
| 能构造出贪心得不到最优解的**反例**   | 0-1 背包：物品 {(60,10), (100,20), (120,30)}，容量 50 |
| 当前选择会**影响后续可选范围**的质量 | 最长公共子序列（LCS）                                 |
| 需要"**回头看**"才能做出正确决策     | 编辑距离                                              |
| 问题具有**重叠子问题**               | 矩阵链乘法                                            |

**经验判断流程**：

```
能否找到反例？
  ├─ 是 → 贪心不可行，考虑 DP / 回溯
  └─ 否 → 尝试证明贪心选择性质
              ├─ 能证明 → 用贪心，复杂度低
              └─ 不能证明 → 保险起见用 DP
```

## 七、一句话总结

贪心的本质就是一个**排序 + 单次遍历**的过程：找到正确的排序准则，扫一遍就能得到最优解，时间复杂度通常是 $O(n \log n)$，代码简洁，效率高——但前提是你能证明"每步选最好的，最终就是全局最好的"。
