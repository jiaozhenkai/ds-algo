# 贪心与二分答案

## 一、什么是"二分答案"

普通二分是在一个**有序数组**里查找某个值。二分答案则是一种**算法思想**:当题目要我们求一个最优解(最大值的最小、最小值的最大、第 k 大等),而**直接构造最优解很难**,但**验证某个候选解是否可行很容易（需要事先知道解集空间）**时,我们就可以在"答案的取值范围"上做二分。

核心在于把一个**最优化问题**转化为一系列**判定问题**:

> 「求满足条件的最大/最小的 $x$」 $\Longrightarrow$ 「给定 $x$,判断它是否可行 `check(x)`」

而贪心常常就出现在这个 `check(x)` 里——用贪心策略在线性时间内判断可行性。

---

## 二、可二分的根本前提:答案的**单调性**

二分答案能成立的唯一充要条件是:**判定函数关于答案具有单调性**。

设判定函数为 $\text{check}: \mathbb{Z} \to \{\text{true}, \text{false}\}$,定义在答案区间 $[L, R]$ 上。

### 情形 A:最小化最大值(求最小的可行 $x$)

如果 $x$ 可行,那么任何 $x' \ge x$ 也可行。即:

$$
\text{check}(x) = \text{true} \implies \forall\, x' \ge x,\ \text{check}(x') = \text{true}
$$

此时 `check` 的真值序列形如 `F F F … F T T T … T`,我们要找**第一个 T** 的位置。

### 情形 B:最大化最小值(求最大的可行 $x$)

如果 $x$ 可行,那么任何 $x' \le x$ 也可行。即:

$$
\text{check}(x) = \text{true} \implies \forall\, x' \le x,\ \text{check}(x') = \text{true}
$$

此时真值序列形如 `T T T … T F F F … F`,我们要找**最后一个 T** 的位置。

> 数学语言:存在一个分界点 $x^\*$,使得 $\text{check}$ 在 $[L, R]$ 上是单调布尔函数。二分答案就是用 $O(\log(R-L))$ 次 `check` 调用定位这个分界点 $x^\*$。

总复杂度 = $O(\log(R-L)) \times O(\text{check 的代价})$。

## 三、两套不会错的二分模板

二分最容易出错的是**边界与死循环**。下面给出两套经过验证、能精确收敛的模板(以整数为例)。

### 模板 1:找第一个满足条件的 $x$(最小化)

```python
# 真值序列: F F F T T T,找第一个 T
def solve_min():
    lo, hi = L, R          # 答案区间 [L, R],闭区间
    while lo < hi:
        mid = (lo + hi) // 2     # 下取整,偏左
        if check(mid):           # mid 可行
            hi = mid             # 答案可能是 mid 或更小,收缩右界
        else:
            lo = mid + 1         # mid 不行,答案在右侧
    return lo                    # lo == hi 即为答案
```

### 模板 2:找最后一个满足条件的 $x$(最大化)

```python
# 真值序列: T T T F F F,找最后一个 T
def solve_max():
    lo, hi = L, R
    while lo < hi:
        mid = (lo + hi + 1) // 2  # 上取整,偏右(防死循环关键!)
        if check(mid):            # mid 可行
            lo = mid              # 答案可能是 mid 或更大,收缩左界
        else:
            hi = mid - 1          # mid 不行,答案在左侧
    return lo
```

**记忆口诀**:
- `lo = mid` 时,`mid` 必须**上取整**(`+1`),否则 `lo=mid` 在 `hi=lo+1` 时永远卡住 → 死循环。
- `hi = mid` 时,`mid` 用**下取整**即可。
- 谁不带 `+1`/`-1`(即 `= mid`),就让 `mid` 往谁那边偏。

### 浮点二分(实数答案)

```python
def solve_float():
    lo, hi = L, R
    for _ in range(100):          # 固定迭代次数,避免精度死循环
        mid = (lo + hi) / 2
        if check(mid):
            hi = mid
        else:
            lo = mid
    return lo
```

用固定迭代次数(如 100 次,精度可达 $2^{-100}$ 级别相对误差)比用 `while hi-lo > eps` 更稳健。

## 四、经典模型详解

### 模型 1:最小化最大值 —— 分段/分配问题

> **抽象**:把一个序列分成若干段(或把任务分给若干工人),要让"最大的那一段/那个人的负担"尽可能小。

**例题(LeetCode 410. Split Array Largest Sum / 洛谷 P1182 数列分段)**:
给定非负整数数组 `nums`,把它分成 `m` 个**连续**子数组,最小化这些子数组和的**最大值**。

**为什么可以二分答案**:设答案(最大子数组和的上限)为 $x$。
- $x$ 越大,允许每段装的越多,需要的段数越少;
- $x$ 越小,需要的段数越多。

即「在限额 $x$ 下所需段数」关于 $x$ **单调不增**。所以「段数 $\le m$」这个判定关于 $x$ 单调,满足情形 A。

**check 用贪心**:给定每段和上限 $x$,从左到右扫描,能塞进当前段就塞,塞不下就**新开一段**(贪心:尽量填满当前段),数最终需要几段。

```python
def splitArray(nums, m):
    def check(cap):
        # 在每段和 <= cap 的前提下,贪心计算需要多少段
        segs, cur = 1, 0
        for v in nums:
            if cur + v > cap:
                segs += 1        # 开新段
                cur = v
            else:
                cur += v
        return segs <= m         # 段数够用即可行

    lo, hi = max(nums), sum(nums)   # 答案下界=最大单元素,上界=全部之和
    while lo < hi:
        mid = (lo + hi) // 2
        if check(mid):              # mid 够大,能在 m 段内完成
            hi = mid
        else:
            lo = mid + 1
    return lo
```

- 答案下界 $L = \max(\text{nums})$:任何一段至少要装得下最大的单个元素。
- 答案上界 $R = \sum(\text{nums})$:全放一段。
- 复杂度 $O(n \log(\sum - \max))$。

**贪心可行性的证明要点**:在限额 $x$ 下,"能不开新段就不开"的贪心所用段数是**最少**的。反证:若存在更优分法用更少段数,则在第一个分歧点处,贪心当前段装的元素是更优解的超集(前缀),后续不会更差。故贪心给出的段数即理论最小段数,用它判定 $\le m$ 正确。

### 模型 2:最大化最小值 —— 间隔/资源问题

> **抽象**:在若干位置中选一部分,让"相邻被选位置的最小间距"尽可能大;或分配资源让"最少的那一份"尽可能多。

**例题(LeetCode 1552. Magnetic Force Between Two Balls / 洛谷 P2678 跳石头 同源)**:
在数轴上有 `n` 个位置,放 `k` 个球,最大化任意两球间的**最小距离**。

**为什么可以二分答案**:设答案(最小间距下限)为 $x$。
- $x$ 越小,越容易放下 $k$ 个球(限制宽松);
- $x$ 越大,越难放下。

即「在最小间距 $\ge x$ 的约束下能放的球数」关于 $x$ **单调不增**,满足情形 B。

**check 用贪心**:给定最小间距 $x$,把位置排序,从最左开始放第一个球,之后**只要距离上一个球 $\ge x$ 就立刻放**(贪心:尽早放,给后面留空间),数能放几个。

```python
def maxDistance(position, k):
    position.sort()
    def check(d):
        # 最小间距 >= d 时,贪心能放几个球
        cnt, last = 1, position[0]
        for p in position[1:]:
            if p - last >= d:
                cnt += 1
                last = p
        return cnt >= k          # 能放够 k 个即可行

    lo, hi = 1, position[-1] - position[0]   # 间距范围
    while lo < hi:
        mid = (lo + hi + 1) // 2   # 最大化,上取整
        if check(mid):             # mid 间距下还能放够 k 个
            lo = mid               # 尝试更大间距
        else:
            hi = mid - 1
    return lo
```

注意这里用的是**模板 2**(最大化,`mid` 上取整,`lo = mid`)。

**贪心可行性**:固定间距下限 $x$,"能放就尽早放"的策略放的球数最多。因为把第一个球放最左、之后贪心地选第一个合法位置,可以证明任何其它放法都不会放得更多(交换论证:把某个球往左挪到贪心位置不会减少后续可放数)。

## 五、识别"二分答案"的信号

遇到下列特征,优先考虑二分答案:

1. 求"**最大值的最小**""**最小值的最大**""第 $k$ 大/小"等极值优化。
2. 直接求解很难,但**给定一个具体答案去验证可行性**很容易(通常 $O(n)$ 贪心)。
3. 答案具有**单调性**:答案放宽则更容易满足,收紧则更难满足。
4. 答案值域是**整数或实数区间**,范围已知或可估计。

**通用解题四步**:
1. 设答案为 $x$,明确要"最大化"还是"最小化"。
2. 写出判定函数 `check(x)`(常用贪心 / 前缀和 / 模拟)。
3. 验证 `check` 关于 $x$ 的单调方向,确定用模板 1 还是模板 2。
4. 确定值域 $[L, R]$,套模板二分。

## 七、一句话总结

> 当题目要求**极值最优解**、**直接构造困难但验证容易**、且答案具有**单调性**时,把"求最优解"转化为"在答案值域上二分 + 贪心验证可行性",即可用 $O(\log V \cdot \text{check})$ 高效求解。最大化最小值用**模板 2(mid 上取整)**,最小化最大值用**模板 1(mid 下取整)**。

# 为什么会有"最大值的最小"和"最小值的最大"

你的疑惑很正常:"最大值"已经是个极值了,怎么还能再求它的最小?关键在于——**这里有两层结构,两个"极值"作用在不同的对象上**。

## 核心:两层极值,作用对象不同

一句话拆解:

> **最大值的最小** = 你有很多种**方案**,每种方案内部会产生一个"最大值";你要在所有方案里挑出那个"最大值最小"的方案。

- **第一层(内层)**:对**单个方案内部**取极值 → 得到这个方案的"最大值"(或最小值)。
- **第二层(外层)**:对**所有可能的方案**取极值 → 在众多方案的结果里挑最优的。

所以两个极值不矛盾,因为它们**不是作用在同一组数上**:

$$
\min_{\text{所有方案 }S}\ \Big(\ \max_{\text{方案 }S\text{ 内部}}\ (\text{某个量})\ \Big)
$$

内层 $\max$ 算的是"一个方案有多糟",外层 $\min$ 是"在所有方案里挑最不糟的"。

---

## 例子一:最大值的最小(分糖果)

把一排糖果 `[7, 2, 5, 10, 8]` 分给 **2 个**小朋友,要求**按顺序连续分**(不能打乱)。每个小朋友拿到一段的糖果总数。

**问题**:怎么分,才能让"拿得最多的那个小朋友"拿得**尽量少**?(也就是分得尽量公平)

试几种分法:

| 分法 | 第一份 | 第二份 | 两份各自的和 | **这个方案的最大值** |
|---|---|---|---|---|
| A | `[7]` | `[2,5,10,8]` | 7, 25 | **25** |
| B | `[7,2]` | `[5,10,8]` | 9, 23 | **23** |
| C | `[7,2,5]` | `[10,8]` | 14, 18 | **18** |
| D | `[7,2,5,10]` | `[8]` | 24, 8 | **24** |

- **内层 max**:每一行最右边那个数,是"这个分法里最累的那份"。
- **外层 min**:在 25、23、18、24 这些"最大值"里,挑最小的 → **18**(方案 C)。

所以答案是 **18**。我们求的是"在所有分法中,那个**最大份**最小的分法",这就是"**最大值的最小**"。

> 直觉:每个分法都有个"最惨的一份(最大值)",我们要让最惨的那份尽可能不惨。

## 例子二:最小值的最大(放路灯)

数轴位置 `[1, 4, 9, 16]` 上放 **2 盏**路灯(灯只能放在这些点上)。每个方案里,两盏灯会有一个**最近的间距**。

**问题**:怎么放,才能让"两盏灯的最近间距"**尽量大**?(放得尽量分散)

试几种放法:

| 放法 | 选的位置 | 间距 | **这个方案的最小间距** |
|---|---|---|---|
| A | `1, 4` | 3 | **3** |
| B | `1, 9` | 8 | **8** |
| C | `1, 16` | 15 | **15** |
| D | `4, 16` | 12 | **12** |

(只放 2 盏时只有一个间距,如果放 3 盏以上,一个方案内就会有多个间距,要取其中**最小**的那个。)

- **内层 min**:每个方案里那个"最挤的一对灯(最小间距)"。
- **外层 max**:在 3、8、15、12 里挑最大的 → **15**(方案 C)。

答案 **15**。我们求的是"在所有放法中,那个**最近间距**最大的放法",这就是"**最小值的最大**"。

> 直觉:每个方案都有个"最挤的地方(最小值)",我们要让最挤的地方尽可能不挤。

---

## 为什么不矛盾——再点一次

| | 内层(对一个方案) | 外层(对所有方案) | 通俗含义 |
|---|---|---|---|
| 最大值的最小 | 取 **max**(方案里最重的负担) | 取 **min** | 让"最坏情况"尽量好(公平分配) |
| 最小值的最大 | 取 **min**(方案里最弱的环节) | 取 **max** | 让"短板"尽量长(资源分散/均衡) |

矛盾感来自于以为两个极值在同一组数上;实际上**内层在"一个方案内部"求极值,外层在"方案与方案之间"求极值**,层次不同,自然能共存。

---

## 和二分答案的联系

回到上一轮的内容:这类问题之所以能用**二分答案**,正是因为外层那个"最优值"具有单调性——

- **最大值的最小**:你猜一个上限 $x$(每份不超过 $x$),$x$ 越大越容易分得开 → 找**最小**的可行 $x$(模板 1)。
- **最小值的最大**:你猜一个下限 $x$(间距至少 $x$),$x$ 越小越容易放得下 → 找**最大**的可行 $x$(模板 2)。

二分干掉的就是"外层"那个 min/max,而"内层"的 max/min 通常在 `check(x)` 里用贪心一遍扫出来。

# abc463_d
**全符合"二分答案 + 贪心"的经典范式**,而且正好是我们前面讲的**「最大化最小值」(最小值的最大)**类型。下面详细拆解。

## 题意速览

数轴上有 $N$ 块布,第 $i$ 块覆盖区间 $[L_i, R_i]$。要选出 $K$ 块**两两不重叠**的布(仅在单点相接也算重叠,见样例 2),使得选中布之间的**两两距离的最小值**尽可能大,输出这个最大值;无法选出则输出 $-1$[[1]](https://atcoder.jp/contests/abc463/tasks/abc463_d)。

其中两块不重叠的布的距离定义为两区间间的最小间隙 $|p-q|$。

## 为什么它是"二分答案 + 贪心"

### 1. 目标形态吻合

题目要求的是"**距离最小值的最大化**"——正是上一轮讲的**最大化最小值(情形 B)**:
- **内层 min**:一个选法内部,取所有相邻布之间最挤的那对距离;
- **外层 max**:在所有合法选法中,让这个"最挤距离"尽可能大。

$$
\max_{\text{合法选 }K\text{ 块}}\ \Big(\ \min_{\text{被选相邻对}}\ \text{距离}\ \Big)
$$

### 2. 单调性成立(可二分的前提)

设答案(距离下限)为 $x$:
- $x$ 越小,约束越松,能选出的两两间距 $\ge x$ 的布越多;
- $x$ 越大,越难凑够 $K$ 块。

即「在"相邻布间距 $\ge x$"约束下能选出的最大布数」关于 $x$ **单调不增**。所以「能否选够 $K$ 块」关于 $x$ 是单调布尔函数,真值序列形如 `T T T … F F F`,用**模板 2(找最后一个 T,mid 上取整)**。

### 3. check(x) 用贪心(带一个关键预处理)

这道题比裸的"跳石头"多了一步——布是**区间**不是点,还可能**互相重叠/包含**。核心贪心思路:

**关键观察**:两块不重叠布之间的距离,只取决于左边那块的**右端点 $R$** 和右边那块的**左端点 $L$**。要让选中的布尽量"靠左、给后面留空间",在同一段位置上应当**优先选右端点 $R$ 最小的布**(右端点越靠左,后面能放的越多——这就是贪心的交换论证)。

因此 check(x) 的贪心是"区间调度 + 跳石头"的结合:

```python
import sys

def solve():
    input_data = sys.stdin.buffer.read().split()
    idx = 0
    N = int(input_data[idx]); idx += 1
    K = int(input_data[idx]); idx += 1
    cloths = []
    for _ in range(N):
        L = int(input_data[idx]); R = int(input_data[idx+1]); idx += 2
        cloths.append((L, R))

    # 关键预处理:按 R 升序排序(区间调度经典贪心序)
    cloths.sort(key=lambda c: c[1])

    def check(x):
        # 判定:在"相邻被选布的距离 >= x"约束下,能否选够 K 块
        cnt = 0
        last_R = -10**18  # 上一块被选布的右端点
        for L, R in cloths:
            # 当前布要能被选,需满足:它的左端点 L 与上一块右端点 last_R 的间距 >= x
            #   即 L - last_R >= x  (注意:>=x,单点相接 x=0 才算,题目单点相接视为重叠)
            if L - last_R >= x:
                cnt += 1
                last_R = R   # 贪心:选它,更新右端点(因已按 R 排序,R 尽量小)
                if cnt >= K:
                    return True
        return cnt >= K

    # 先判无解:x=1(任意两块严格不重叠,间距至少 1)都选不够 K 块 -> -1
    # 因单点相接算重叠,所以合法需 L - last_R >= 1
    if not check(1):
        print(-1)
        return

    # 二分答案:最大化最小距离,模板 2
    lo, hi = 1, 10**9
    while lo < hi:
        mid = (lo + hi + 1) // 2      # 上取整,防死循环
        if check(mid):
            lo = mid
        else:
            hi = mid - 1
    print(lo)

solve()
```

> 说明:因为题目规定"单点相接也算重叠",所以合法相邻需要 $L - \text{last\_R} \ge 1$(严格分开);而"score $\ge x$"要求间距 $\ge x$。判无解时用 `check(1)`——连严格分开选 $K$ 块都做不到就输出 $-1$(对应样例 2)。
> 复杂度:排序 $O(N\log N)$ + 二分 $O(\log 10^9)$ 次、每次 check $O(N)$ = $O(N\log N + N\log V)$,$N \le 2\times10^5$ 完全够。

## 结论

| 判定项 | 是否满足 |
|---|---|
| 求极值最优解(距离最小值的最大化) | ✅ 最大化最小值 |
| 答案具单调性 | ✅ 间距 $x$ 越大越难凑够 $K$ 块 |
| 直接构造难、验证易 | ✅ check(x) 用贪心 $O(N)$ 判定 |
| check 内部用贪心 | ✅ 按 $R$ 排序 + 跳石头式贪心 |

**ABC463 D 是标准的"二分答案(最大化最小值)+ 贪心 check"题**,是练这个套路的很好例题。相比纯"跳石头",它的加分点在于:对象是**区间**,需要先用"**按右端点排序的区间调度贪心**"把区间转化为可跳的序列。

# 如果不用"二分答案 + 贪心",难点在哪里

回到 ABC463 D 这道题:选 $K$ 块两两不重叠的布,最大化它们之间的**最小距离**。抛开"二分 + 贪心"这个套路,直接硬做会撞上几堵墙。逐个说清楚。

---

## 难点一:直接求"最优解"没有明确的贪心方向

你可能会想:能不能一次扫描就贪心地选出最优的 $K$ 块?**不行。**

问题的本质矛盾是:**这里有两个互相冲突的目标同时要满足**——

1. 要选够 **$K$ 块**(数量约束);
2. 要让相邻间距的**最小值最大**(质量约束)。

如果只优化数量(尽量多选、能选就选),得到的间距最小值往往很小;如果只优化间距(每次跳得尽量远),又可能选不够 $K$ 块。

**关键症结**:在不知道"目标间距是多少"之前,你**无法判断当前这块布该不该选**。
- 若目标间距是 2,某块布离上一块距离 3,该选;
- 若目标间距是 5,同样这块布就**不能**选,得跳过去等更远的。

也就是说,**"选或不选"这个决策,依赖于那个你还没求出来的答案本身**。这就是死结——没有二分把答案先"猜"出来,贪心根本无从下手。

> 二分答案的精妙正在于此:它先**假设**答案是 $x$,把"最优化"降维成"给定 $x$ 判可行",这时贪心方向瞬间清晰(间距够 $x$ 就选)。

---

## 难点二:朴素枚举答案会超时

既然贪心需要先知道目标间距 $x$,那**从小到大枚举** $x$ 行不行?

$$
x = 1, 2, 3, \dots, 10^9,\quad \text{每个 } x \text{ 跑一遍 } O(N) \text{ 的 check}
$$

这就是"**线性扫描答案**"而非"二分答案"。问题:$x$ 的值域是 $[1, 10^9]$,复杂度 $O(V \cdot N) = 10^9 \times 2\times10^5$,**天文数字,必然 TLE**。

正因为 check 关于 $x$ 有**单调性**(见上一轮),我们才能把 $O(V)$ 次枚举压成 $O(\log V) \approx 30$ 次二分。**去掉二分,只留贪心 check,复杂度就爆了。**

---

## 难点三:直接暴力搜索"选哪 K 块"是组合爆炸

如果连贪心方向都放弃,老老实实枚举"从 $N$ 块里选 $K$ 块"的所有组合,再算每个组合的最小距离取最大:

$$
\binom{N}{K} \text{ 种组合},\quad N,K \le 2\times10^5
$$

这是**指数级**规模,$\binom{2\times10^5}{5}$ 就已经是约 $10^{26}$ 量级,彻底不可行。

---

## 难点四:换成 DP 也不轻松,而且状态里绕不开"距离"

有人会想:区间选择问题不是常用 DP 吗?比如设 $f[i][j]$ = 前 $i$ 块布选了 $j$ 块时的某个最优值。难点在于:

- **DP 要优化的量是"最小间距"**,而最小间距是一个"瓶颈型"目标(取 min 再取 max),它**不满足 DP 所需的最优子结构叠加性**——前面选法的"局部最小间距"和后面的决策耦合,状态转移时你需要同时记录"已选块数"和"上一块位置",还要在转移里维护 min,状态设计和转移都很别扭。
- 就算勉强设出 $f[i][j]$,状态数是 $O(N \cdot K)$,$N,K$ 同阶时是 $O(N^2) = 4\times10^{10}$,同样超时。

**瓶颈型最优化(max-min / min-max)问题,天生更适合二分答案,而不是直接 DP**——这正是二分答案套路存在的意义。

---

## 难点五:区间的"重叠/包含"让贪心序也不显然

即使有了二分把问题变成判定,这道题相比裸的"跳石头"还多一层坑:对象是**区间**而非点,且区间可以**互相包含、部分重叠**。

- 点的场景里,"从左到右能跳就跳"天然正确;
- 区间场景里,若不做正确排序,贪心会选错。**必须先按右端点 $R$ 升序排序**(区间调度经典贪心),保证每次选的布"右端点尽量靠左",给后面留最大空间。这一步的正确性需要**交换论证**来证明,不是拍脑袋就对。

换句话说,不用套路直接想,连"该按什么顺序处理区间"都不显然,容易写出看似合理实则错误的贪心。

---

## 一句话总结

| 不用二分+贪心的做法 | 撞的墙 |
|---|---|
| 一次扫描直接贪心选 | 决策依赖未知的答案,方向不定 |
| 线性枚举答案 $x$ | $O(V\cdot N)=10^{14}$,TLE |
| 暴力枚举 $\binom{N}{K}$ 组合 | 组合爆炸,指数级 |
| 直接 DP | 瓶颈型目标难叠加 + $O(NK)$ 超时 |

> **根本难点**:这是一个**瓶颈型(max-min)最优化**问题,最优解的"选/不选"决策与答案本身循环依赖,导致无法直接贪心、直接枚举又会超时。二分答案恰好斩断这个循环——先假设答案再用单调性 + 贪心快速验证,把指数/线性级的优化问题压到 $O(N\log N)$。
